# AI Document Ingest

Document ingest at `/ingest` is the fastest way to get a paper contract into LapseIQ: drop a PDF, the AI extracts every field, you review and approve.

## What you'll see

**Upload area** at the top — drag-and-drop or click to select files. Accepts PDF, Word documents, plain text, images (TIFF/JPEG/PNG), and `.lic` license files. One file at a time or a batch.

**Review queue** below — every file in flight. Click into a queue entry to see the extracted fields side-by-side with the source document.

**Per-field confidence color-coding** — high-confidence fields are green, medium yellow, low red. Focus your review on the red ones first.

## How the extraction pipeline works

1. You drop a file. Server validates the MIME type against a magic-byte signature (so a renamed `.exe` posing as a `.pdf` is rejected even if its Content-Type header is forged).
2. The extractor pulls text from the file. PDFs go through `pdf-parse`. Word docs through `mammoth`. Plain text and `.lic` files are decoded directly. Images skip text extraction and go straight to vision.
3. The text or image is sent to the configured AI provider with a strict JSON schema.
4. The AI returns extracted fields with per-field confidence scores. Malformed responses are rejected (not silently passed through).
5. An IngestionSession row is created. You review the extracted fields in the UI.
6. On approve, the document is linked to the contract record — created new or updated if a definitive vendor match exists.

The AI is instructed to extract **only what's clearly stated**. If a field isn't present or unclear, it returns null. False data is worse than missing data when you're going to review the result anyway.

## What gets extracted

Vendor name, product, contract numbers (contract #, customer #, PO, invoice #), quantity, cost per license, start/end dates, auto-renewal terms (yes/no, notice days, escalation clause), PO/invoice numbers, department, requestor, delivery method, notes, vendor support contacts, reseller info, vendor contacts, clause flags (multi-year, minimum commit, true-up).

That's ~30 fields. The schema is defined in `server/lib/extractor.js::EXTRACTION_SCHEMA`. Custom contract fields are NOT auto-extracted — those require manual entry or a post-approve update.

## Vendor matching during ingest

The same normalizer that powers manual vendor creation runs here too. If the AI returns "Microsoft Corporation" and you already have a "Microsoft" vendor, they're matched automatically. Near-matches prompt for confirmation. No match creates a new vendor.

## Common workflows

**"I have 50 PDFs from procurement to import."** Drop them all into ingest in a batch. The queue lets you walk through them one at a time. High-confidence rows can be batch-approved (Settings → AI → "Auto-approve threshold" sets the bar).

**"The AI got the vendor wrong."** Click into the queue entry. The vendor field has an autocomplete dropdown — pick the right one. Approve. Future contracts from this vendor with similar shapes train the system implicitly (the alias map can be updated to capture variants).

**"The AI extracted a custom field name but I want it mapped to ours."** Custom fields aren't extracted by the AI. Edit the contract after approve and fill in custom fields manually, or use the CSV import path for repeatable column mappings.

**"I uploaded a Word doc and the dates look wrong."** Mammoth (the Word parser) sometimes mis-formats embedded tables. Re-save the Word file as PDF and re-upload. PDF text extraction is more reliable for table-heavy contracts.

**"How much does each extraction cost?"** Demo mode: hard-capped at 2 per user per day. Self-hosted: unlimited by default; operators can set `AI_DAILY_CAP_PER_USER=N` in `.env` for a soft cap.

## When something looks wrong

**Upload rejected with "MIME type not allowed".** The magic-byte signature didn't match. Renamed file? Re-save in the intended format and retry.

**Extraction returned mostly nulls.** The source document might be a scanned image PDF without OCR. Run it through an OCR tool first, then re-upload.

**Confidence is uniformly red.** The contract is in a non-English language, or it's an unusual format the AI hasn't seen much of. Pick a comparable contract you've already approved and manually copy values; the system gets better at that vendor's format over time as your alias map grows.

**Got a 402 "You've used N/N of your daily AI calls".** Demo mode quota (2/day). Resets at midnight UTC. Self-hosted: bump or remove `AI_DAILY_CAP_PER_USER` in `.env`.
