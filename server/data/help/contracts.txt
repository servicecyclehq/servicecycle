# Contracts

A Contract is the operating unit of LapseIQ. Everything else — vendors, alerts, briefs, savings, reports — orbits around contracts.

## What you'll see

**Contract list** at `/contracts`: every active contract you have access to, filterable and sortable. Filters: status (active/under-review/renewed/cancelled/expired), vendor, end-date window, owner, category, has-PO, evaluation window.

**Contract detail** at `/contracts/:id`: every field on the record, plus the Renewal Checklist, Documents panel, Communications log, Purchase Orders, and (when AI is on) the renewal brief.

**Archived contracts** at `/contracts/archived`: contracts that have been soft-deleted with `archivedAt`. Original data preserved, hidden from the default list. Admin can unarchive.

## The three dates that matter

Every contract has three different "renewal-shaped" dates. Mixing them up is the most common new-user mistake.

**End date.** When the contract expires. The renewal date.

**Evaluation-start-by date** (shown in the UI as "review by"). Auto-calculated from end date and contract value: $100K+ → 180 days back, $25K-$100K → 90 days back, under $25K → 30 days back (no cost data → 60 days). Higher-value contracts need a longer evaluation runway because they involve more stakeholders. This is when you should *start* working the renewal. Those tiers are defaults — an admin can change the value breakpoints and the lead-time days in Settings → General → Evaluation Lead Times.

**Cancel-by date.** Only set on contracts with auto-renewal enabled. It's end date minus the notice period. Miss this and the contract auto-renews — often at an undisclosed price uplift. The Auto-Renewal Traps panel on the dashboard exists specifically to surface contracts approaching this date.

Practitioner shorthand: review-by is when you start, cancel-by is when the door closes, end date is when the bill comes either way.

## Urgency colors

LapseIQ colors each active contract by how close it is to its review-by (Evaluate-By) date, using one shared model so the contract header, the Key Dates chip, and the day chips on the contract list all agree:

- **Green** — before the Evaluate-By date. On track; nothing to do yet.
- **Amber** — at or past the Evaluate-By date but still more than 30 days out. The window is open — start working the renewal.
- **Red** — 30 days or fewer to the end date, or already past it (overdue).

Renewed, cancelled, and expired contracts show neutral. Because the Evaluate-By date is value-tiered, retuning the tiers in Settings → General → Evaluation Lead Times shifts when a contract turns amber.

## How the data lands here

Three ways data gets into a contract: (1) manual entry via the new-contract form at `/contracts/new`, (2) AI document ingestion at `/ingest` — drop a PDF, the AI extracts fields for review, (3) CSV import from the Settings → Imports tab.

When you change `endDate`, `costPerLicense`, `quantity`, or `autoRenewalNoticeDays`, the auto-calculated dates (evaluation-start-by, cancel-by) recompute automatically.

## Common workflows

**"I need to bulk-update 12 contracts to set the same owner."** Select them on the list page (the checkboxes in the row header), then use the sticky toolbar that appears at the bottom: Set Owner, Change Status, Archive/Restore.

**"I need to find every contract for Microsoft."** Use the Vendor filter dropdown. Vendor normalization is automatic — "Microsoft Corporation", "Microsoft", and "MS" map to one vendor record (configurable via aliases in Settings → Vendors).

**"I need a CSV of every contract ending in Q3."** Filter by end-date window (next 90 days or whatever), then click Export. The CSV includes every column you see on the list plus custom fields.

**"This contract was renewed — how do I link the new one to it?"** On the old contract, set status to `renewed`. Then on the new contract, set the `parentContractId` to the old one's id. The two are now chained and the budget/spend reports follow the chain.

## When something looks wrong

**The cancel-by date is wrong.** It's recomputed any time you edit end date, auto-renewal flag, or notice days. If it still looks off, check that you've set `autoRenewalNoticeDays` correctly — the contract's literal language might say 30, 60, 90, or 180 days.

**The review-by date is too far out / too close in.** Review-by is tiered by total contract value. Check `costPerLicense × quantity`. A contract with missing cost data defaults to a 60-day review window. If the whole account's windows look off, an admin may have changed the tiers in Settings → General → Evaluation Lead Times.

**Bulk-update didn't catch some contracts.** Scope restriction. A viewer with `contractScopeRestricted` can only mutate contracts they own. Check the user's permissions.

**Contract auto-flipped to expired and I don't think it should have.** End-date-in-past + status active triggers an auto-expire when the dashboard loads. If you want to keep an expired-but-active contract visible, set status to `under_review` instead.
