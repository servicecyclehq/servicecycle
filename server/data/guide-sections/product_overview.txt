# ServiceCycle at a glance + the renewal cycle

ServiceCycle is self-hosted software for managing software contract renewals, license utilization, and negotiated savings. Built by ForgeRift LLC. Runs as Docker containers on the customer's own infrastructure. The customer brings their own PostgreSQL database, their own document storage, and their own (optional) AI provider key. There is no ServiceCycle cloud.

Three product principles:

- **Self-hosted and local-first.** Every contract, every vendor record, every uploaded document stays inside the customer's network. The default `docker-compose.yml` brings up the API, the React frontend, and PostgreSQL — all on hardware the customer controls.
- **Telemetry-free.** A default install with no integrations enabled makes zero outbound calls. There's no analytics beacon, no license-validation callback, no anonymous-usage ping. When the customer turns on the AI provider, Resend, Slack/Teams webhooks, or any S3-compatible storage, those calls go directly from the customer's box to the provider — never through ForgeRift.
- **Practitioner-built.** The data model came from someone who has actually run renewals for years. Things like the separate `cancelByDate` (auto-calculated from notice period) versus `endDate`, the tiered review-by horizon based on contract value, the savings tracker, the renewal checklist — these aren't generic CLM features.

## High-level data model

Schema lives in `server/prisma/schema.prisma`.

- **Account** — the tenant boundary. One per customer install, or one per visitor in the demo sandbox. Every other row is scoped to an `accountId`.
- **Users** — belong to an Account; role is `admin`, `manager`, `viewer`, or `consultant`.
- **Vendors** — the organizations the customer buys software from. Co-term complexity, support contacts, the budget uplift %, and the curated alias list live here.
- **Contracts** — belong to an Account, reference a Vendor. Carry the 30+ field normalized schema (identity, financials, dates, auto-renewal terms, ownership, savings tracking, license utilization, custom fields). Most of the work happens here.
- **Documents** — uploaded files attached to a contract or held at the account level. Storage backend is pluggable — local filesystem by default, or any S3-compatible store.
- **Alerts** — time-windowed reminders. `review_by`, `cancel_by`, `renewal`, plus billing alerts. Generated by `server/lib/alertEngine.js`.
- **ActivityLog** — every meaningful change. Append-only feed.

About thirty more models exist (`IngestionSession`, `ConsultantAccess`, `VendorNews`, `CustomFieldDefinition`, `RefreshToken`, `BackupLog`, …) that you'll learn as needed. The seven above are what most questions anchor on.

## The renewal cycle, end-to-end

Most product questions come from someone in the middle of one of these steps. When a user asks "where do I do X," locate them in this cycle first, then point to the screen.

1. **Inventory.** Drop documents into `/ingest`, AI extracts fields, user reviews and approves. Vendors are normalized via `server/lib/vendorNormalizer.js` (handles the "Microsoft Corporation / Microsoft / MSFT" problem). Custom fields cover whatever the standard schema doesn't.
2. **Configuration.** Per-vendor uplift defaults, per-user alert preferences, fiscal year start month, optional Slack/Teams webhooks, storage backend, AI provider. Most of this lives in `/settings`. Power-user defaults: `server/.env.example`.
3. **Watching.** As contracts approach their `evaluationStartByDate`, alerts fire — daily digest emails at 07:00 server time, optional Slack/Teams pings, in-app notification bell. Dashboard at `/dashboard` shows what needs attention today. Calendar view (12-month forward) shows review-by, cancel-by, and renewal dates color-coded distinctly.
4. **Negotiating.** A contract enters Under Review (auto-populated when someone opens the renewal checklist or manually flipped via status). Renewal brief generator at `/contracts/:id` summarizes the contract and surfaces the negotiation log. Savings tracker captures original ask vs final price.
5. **Closing.** 5-stage checklist (Notice Sent → Proposal Received → Under Negotiation → Awaiting Signature → Signed). Each step writes to the activity log with attribution. When signed, contract transitions to `renewed`; a chained successor is created via `parentContractId`.
6. **Reporting.** Budget Forecast at `/budget` is forward-looking. Executive Spend Report at `/reports/executive-spend` is backward-looking actuals, fiscal-year anchored, manager/admin only. Both exportable.
