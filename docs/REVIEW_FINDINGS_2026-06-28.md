# Review Findings — 06‑27/06‑28 additions (Revenue Intelligence + Document layer)

Read-only audit by four parallel Opus reviewers (security, server bugs/data, client UI/a11y, document liability) over commit range `1388092..HEAD`. Findings only — no code was changed. Demo verified live earlier the same day.

## Verdict

**No P0s. No security breach, no tenant-isolation hole, no liability gap.** The new surfaces are solid. The work to do is a short list of real-but-contained bugs (P1), plus hardening / correctness / a11y / acquisition-copy polish (P2). One scary-looking item — a truncated `documents.ts` in the working tree — was a transient mid-write race during the parallel run; the file is verified intact (matches HEAD) and unmodified.

## Verified clean (diligence positives)

- **Tenant isolation** holds on every new route — `accountId` is always the hard filter; the library `where:any` never drops it; foreign `siteId`/`assetId` just return zero rows.
- **super_admin gating** on Revenue Intelligence is correct (regular admin/manager → 403); cross-tenant `$queryRaw` uses static tagged templates, no SQLi.
- **field_tech default-deny** holds; field writes re-check assignment scope + accountId; archived-asset docs excluded.
- **storage**: path-traversal containment guard, presigned-URL TTL clamp, forced attachment/nosniff. **docCrypto**: AES‑256‑GCM, per-doc HKDF key, constant-time key compare, no silent decrypt passthrough.
- **Migrations** match `schema.prisma` exactly; both new NOT NULL columns carry constant defaults → metadata-only, cannot fail/rewrite existing prod rows. **Seed is idempotent** (no dupes on reseed). No stray scratch files at HEAD.
- **Disclaimers**: upload + download disclaimers render and are acknowledge-gated (non-bypassable) on all three surfaces. **Provenance** defaults `unverified`; `pe_sealed` is manager-gated + audit-logged; nothing silently upgrades trust. One-line PDFs are watermarked DEMONSTRATION / AS‑BUILT / "not a PE‑sealed document" and seeded `as_built`. The one-line **generation engine is genuinely deferred** (no shipped route emits one).

## P1 — real bugs, fix before the next live demo

1. **Field cross-asset work-order state bleed** — `client/src/pages/field/FieldAsset.jsx:470` + route `client/src/App.jsx:316`. The field-asset route has no `key={id}`, and the WO auto-select effect keys only on `[openWOs.length]`. Navigating between two field assets in-app reuses the component without resetting state, so a recorded NETA measurement can POST to the **previous asset's** work order (and the tech can't see the stale selection when each asset has one WO). Data-integrity defect on the compliance-critical path. **Fix:** add `key={id}` to the route (forces remount) and/or reset measurement/deficiency/OCR state on `id` change.

2. **Silent failures on document download / open / reclassify** — `client/src/components/AssetDocumentsCard.jsx` (open/delete/type-change catches) and `client/src/pages/DocumentsLibrary.jsx:84`. Errors are swallowed; the button just stops spinning with no message. The most common real failure (decrypt/MASTER_KEY → 500 with a real message) looks like a dead button. **Fix:** surface a Toast on these catches (component already uses Toast).

3. **Latent field-download IDOR (defensive)** — `server/routes/fieldRoutes.ts:340` (and the matching `/asset/:id` list + `documents.ts` site branch). The doc union does `OR:[{assetId},{siteId: reqAsset.siteId}]`. Safe **today** because `Asset.siteId` is NOT NULL — but if that ever becomes nullable (several models already are), `{siteId:null}` becomes "siteId IS NULL" and a field_tech on a siteless asset would see every account doc with a null siteId. **Fix:** only push the `{siteId}` clause when `reqAsset.siteId` is truthy.

## P2 — hardening, correctness, a11y, acquisition copy

4. **Revenue Intelligence defense-in-depth** — `server/index.ts` mounts the router with only `authenticateToken`; super_admin is enforced per-route (correct today). One future route added without the guard = full cross-tenant PII breach. **Fix:** mount behind `requireSuperAdmin` (dedicated prefix), and **audit-log the `/opportunities` read** (a super_admin reading every tenant's contact book currently leaves no trail). `server/routes/adminOpportunities.ts`.

5. **Demo one-line PDFs imply SC authored them** *(founder's #1 concern)* — `server/scripts/demo-assets/*.pdf` read "AUTO‑GENERATED FROM ASSET RECORDS / Generated by ServiceCycle," contradicting the "SC never generates documents" posture. Mitigated by the DEMONSTRATION watermark + "not a PE‑sealed document" line, but a diligence reader could conclude the generator ships. **Fix:** soften the PDF chrome to "uploaded drawing, rendered for demonstration" / source-attributed.

6. **studyEstimate inverted range** — `server/routes/adminOpportunities.ts:133`. If a rate sheet is saved with `min > max`, the estimate renders "$X–$Y" with X>Y on the demo. **Fix:** validate `min <= max` in the rate-sheet PUT (or clamp).

7. **RateSheet input silently clears on bad value** — `client/src/components/settings/RateSheetSection.jsx:31`. Non-numeric input → `NaN` → serialized `null` → wipes the field with no warning. **Fix:** `Number.isFinite` guard + inline validation.

8. **"Reject Selected" is local-only** — `client/src/pages/OpportunitiesPage.jsx:296`. Rejections (and the captured reason dropdown) never hit an API and vanish on refresh, but the row disappears so the user thinks it persisted. **Fix:** persist it, or relabel "Hide for this session" and drop the dead reason selector.

9. **Export modal not keyboard-accessible** — `client/src/pages/OpportunitiesPage.jsx:505`. Bespoke modal has no `role=dialog`/Escape/backdrop-close/focus-trap. **Fix:** route through the existing `useConfirm()`/ConfirmDialog.

10. **Field disclaimer is a hardcoded copy** — `client/src/pages/field/FieldAsset.jsx:317` duplicates `DOWNLOAD_DISCLAIMER` instead of importing from `client/src/lib/documentDisclaimer.js`. Identical today; drifts when counsel revises wording. **Fix:** import the shared string.

11. **Provenance logged only on `pe_sealed`** — `server/routes/documents.ts:497`. Downgrades / other transitions aren't audited. **Fix:** log all provenance changes (`document_provenance_changed {from,to}`), keep the dedicated attested event for sealed.

12. **Unvalidated `docType` filter** — `server/routes/documents.ts:280`. Invalid enum → Prisma throws → generic 500 (no leak/injection). **Fix:** validate against the allowlist → 400.

13. **Field/UX a11y cluster** — empty field Documents section renders nothing (no "no documents yet" affordance); desktop upload input has no `accept`/size guard (field inputs do); row checkboxes share an identical `aria-label` (add the account name); field download sheet lacks dialog semantics (the gate itself is sound). `FieldAsset.jsx`, `AssetDocumentsCard.jsx`, `OpportunitiesPage.jsx`.

14. **Muddy combined filter** — `server/routes/documents.ts:377`. Passing both `siteId` and `assetId` produces an ambiguous OR (UI never does this). **Fix:** make `assetId` take precedence.

## Recommended fix order

1. **P1 batch** (highest value, all small): field `key={id}` state-bleed fix; Toast on the swallowed download/reclassify errors; guard the `{siteId}` union clause. These are the only items that can corrupt data or look broken in a demo.
2. **Liability copy (P2 #5)** — soften the one-line PDF wording; cheap, removes the one ambiguous acquisition-narrative spot.
3. **Revenue Intelligence hardening (P2 #4)** — mount-level `requireSuperAdmin` + audit the read.
4. **Remaining P2** — rate-sheet validation, reject-persistence/relabel, modal a11y, shared disclaimer import, provenance logging, docType validation, a11y cluster.
5. **P3** — sweep opportunistically.

Fixes should be applied serially (not parallel fix-agents — big-file splice corruption risk) and verified with tsc + client build before deploy, with a version bump (`npm run bump`) on the deploy.
