## What shipped (commit → live)

1. **#2 — Evidence-to-requirement trace map + gap detector** — `15eb5ef`
   - `lib/evidenceTrace.ts`: every active schedule (asset × task) is a 70B
     requirement; its latest completed work order is the evidence. Four tiers:
     **documented / stale (overdue) / undocumented (claimed, no test on file) /
     missing**. `buildAssetEvidenceTrace` (per asset) + `buildEvidenceGapSummary`
     (account/site roll-up: documented %, most under-evidenced test types, assets
     with the biggest gaps).
   - `GET /api/compliance/evidence-gaps` + `/asset-evidence/:assetId`.
   - `EvidenceGapCard` on the compliance report page; `AssetEvidenceTraceCard` on
     the asset detail page.

2. **#5 — Multi-year scope / proposal builder (repair/replace/defer)** — `5c66950`
   - `lib/proposalBuilder.ts`: one line item per asset needing action, classified
     repair / replace / defer with a cost range + funding year, packaged into
     three options (Essential / Recommended / Comprehensive). Reuses the rate
     resolver, repairCostEstimate, and the RUL modernization buckets.
   - `lib/proposalPdf.ts`: a co-branded, sellable PDF.
   - `routes/proposals.ts`: `GET /api/proposals` (JSON) + `/proposal.pdf`
     (manager+); optional `?accountId` lets an **oem_admin build a proposal FOR a
     customer in their partner org** (same partnerOrg wall as the fleet routes).
   - `ProposalCard` on the compliance report page (manager+); **"Proposal PDF"
     button** on the Fleet account drill-down (contractor, per customer).

3. **#4 — Repeat-failure / compliance-drift detector** — `b4918ab`
   - `lib/driftDetector.ts`: scans 24 months for **worsening_trend** (open
     "trending …% since last test" ADVISORY flags → shorten the interval),
     **unclosed_corrective** (a finding open >120d that predates the latest
     completed maintenance → open a corrective WO / escalate), and
     **repeat_failure** (≥3 deficiencies in the window → review the procedure /
     consider replacement). Recommends a **program change, not another ticket.**
   - `GET /api/compliance/drift?siteId=`.
   - `DriftDetectorCard` on the compliance report page.

The compliance report page now stacks: Maturity → Debt Ledger → Proposal →
Evidence Coverage → Drift → Change Brief → Access Blockers (+ Path-to-100). The
asset page gained the per-asset evidence trace.
